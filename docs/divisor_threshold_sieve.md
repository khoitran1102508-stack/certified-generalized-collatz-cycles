# Divisor-threshold prefix sieve: mathematical contract

## 1. Scope

This document specifies divisor-threshold sieve. It is a coding contract, not a novelty claim.
The implementation must preserve exact integer arithmetic and must not change
any definitions in `specification/mathematics.md`.

For fixed integers

- `r >= 1`,
- `A >= A_min_for_period(r)`, and
- `C_max >= 1`,

write

\[
D=2^A-3^r>0.
\]

For a positive composition

\[
a=(a_1,\ldots,a_r),\qquad \sum_i a_i=A,
\]

let

\[
C_r(a)=\sum_{j=0}^{r-1}3^{r-1-j}2^{a_1+\cdots+a_j}.
\]

The normalized parameter generated by `a` is

\[
c(a)=\frac{D}{\gcd(D,C_r(a))}.
\]

The search retains only primitive canonical necklaces with `c(a) <= C_max`.

## 2. Eligible divisors

Define

\[
E(D,C_{\max})=\{g:g\mid D,\ D/g\le C_{\max}\}.
\]

Then

\[
c(a)\le C_{\max}
\iff
\exists g\in E(D,C_{\max})\text{ such that }g\mid C_r(a).
\]

Proof: `c(a)=D/gcd(D,C_r(a))`. The forward direction chooses
`g=gcd(D,C_r(a))`; the reverse direction uses `g <= gcd(D,C_r(a))`.

This equivalence is the only allowed arithmetic basis for the divisor-threshold sieve
parameter-threshold pruning rule.

## 3. Prefix decomposition

Suppose a prefix of length `t` has been selected:

\[
p=(a_1,\ldots,a_t),\qquad
S=\sum_{i=1}^t a_i.
\]

Let

\[
P=\sum_{j=0}^{t-1}3^{r-1-j}2^{a_1+\cdots+a_j},
\]

with `P=0` when `t=0`. Let

\[
m=r-t,\qquad B=A-S.
\]

For every positive suffix composition

\[
b=(b_1,\ldots,b_m),\qquad \sum_i b_i=B,
\]

we have the exact decomposition

\[
C_r(p\Vert b)=P+2^S C_m(b).
\]

A prefix is feasible only when `m >= 0`, `B >= m`, and all selected parts
are positive.

## 4. Reachable suffix residues

For odd modulus `g`, define

\[
R(m,B;g)=\{C_m(b)\bmod g:
 b\text{ is a positive composition of }B\text{ into }m\text{ parts}\}.
\]

The denominator `D` is odd, so every eligible divisor `g` is odd and
`2^S` is invertible modulo `g`.

Base case:

\[
R(1,B;g)=\{1\}\qquad(B\ge1),
\]

because `C_1((B))=1`.

Recurrence for `m >= 2`:

\[
C_m(b_1,\ldots,b_m)
=3^{m-1}+2^{b_1}C_{m-1}(b_2,\ldots,b_m),
\]

and therefore

\[
R(m,B;g)=
\bigcup_{x=1}^{B-m+1}
\left\{
3^{m-1}+2^x y\pmod g:
 y\in R(m-1,B-x;g)
\right\}.
\]

The implementation must memoize this exact set by `(m, B, g)`.
No probabilistic structure, Bloom filter, approximate set, or residue sample is
allowed in the reference implementation.

## 5. Sound prefix rejection rule

For a prefix state `(t,S,P)`, an eligible divisor `g` can still be satisfied
iff

\[
-P\,2^{-S}\pmod g\in R(m,B;g).
\]

The prefix may be pruned iff this membership fails for every
`g in E(D,C_max)`.

### Soundness proposition

If the prefix is pruned by the rule above, no positive suffix completion of
that prefix can generate a normalized parameter `c <= C_max`.

The code documentation must include the proof in two steps:

1. any retained completion would be divisible by at least one eligible `g`;
2. the exact prefix decomposition would force the target suffix residue to lie
   in `R(m,B;g)`, contradicting the failed membership test.

## 6. Search semantics

The divisor-threshold sieve reference search may traverse all positive-composition prefixes.
Canonical-rotation and primitive-period filtering may occur at leaves. This is
intentional: the first objective is to validate the arithmetic sieve without
mixing it with an unproved prefix-necklace canonicalization rule.

A later optimization may add canonical-prefix pruning only under a separate
specification and separate tests.

At a complete leaf:

1. compute `C_r(a)` directly;
2. compute `c=D/gcd(D,C_r(a))`;
3. verify `c <= C_max`;
4. reject noncanonical rotations;
5. reject nonprimitive valuation vectors;
6. construct and independently verify the normalized cycle using `core.py`.

## 7. Required counters

For every `(r,A)` row, record at least:

- total theoretical leaves `comb(A-1,r-1)`;
- recursive nodes visited;
- nodes pruned by divisor-threshold residue test;
- complete leaves reached;
- canonical leaves;
- primitive necklaces;
- retained normalized cycles;
- elapsed seconds;
- DP cache entries;
- maximum reachable-residue set size.

Counters must be deterministic except elapsed time.
